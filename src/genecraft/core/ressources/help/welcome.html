Welcome to GeneCraft

This application is based on the GeneCraft Core library and offers you an user interface to create, simulate and evolve virtual creatures.

This software mainly consists of three modes available on left :

  • Designer mode is used to create entities and see their anatomy. In this mode, start by select a limb by clicking on it in the render view.
  • Experiment mode is used to create experiments and see their results. Here you can also load expermients and creatures from the community database.
  • Theater mode is only used to see experiments in a widescreen.
Notice that you can also show or hide each inspector by right-clicking on the tools bars.

Quick help

The center of this windows is the rendering widget, it contains the world and all creatures you simulate.
You can navigate inside with your keyboard and your mouse. Use AWSD (or arrows) to straf and move the camera. Use drag and drop to rotate the camera.

Where to start

If you don't know what to do first, we invite you to go to the experiment mode and refresh online data from the ressource manager. That will retrieve experiments and creatures stored in the online database. Then you just have to double click on whatever you want to load it.
Loading an experiment will refresh results computed from workers accross the world, you can see those in the "Experiment" widget. Again double click on one element will load it as it was simulated in the experiment.

Your own experiment

If you want to design you own experiment, simply create a new one and then go through the tabs of the experiment properties editor and verify informations about mutations, fitness function, etc.. When you are happy with thoses settings and you have saved your experiment you can launch a worker on it. That will create a "popup" windows where you can enter informations about number of generations, population size, etc.. When you launch it, it'll start computing results and saving the bests one. You can see the bests found results in the "Experiment" widget. For commodity purpose there is an auto-refresh mode.

Warning about online ressources

For this project determinism in the physical engine is very important. It is garanteed in a local environment. But it's very difficult to get across multiples platforms and compilators. So some results computed on others configurations than yours could perform in a very different way in your environment. Please keep that in mind when playing with online results or experiments.

A good way to outpass this problem for a given experiment, is to take the bests results and use them as a seed for a local experiment. Within some generation your worker should be able to change the creature to perform in the same scale on your own computer.
